SSH 通讯

  • 安装
  • ssh 超时连接中断

安装

  1. 首先用密码登陆到你打算使用密钥登陆的用户(假设服务器的用户名为: 小红帽), 然后创建密钥对

    执行上面的命令后,在 ~/.ssh/ 目录下会生成小红帽的 私钥 id_rsa 和公钥 id_rsa.pub

  2. 在服务器上批准使用小红帽的公钥

  3. 设置 SSH, 打开密钥登录功能 (需要 root 权限)

  4. 必须重启 ssh 服务,使配置生效.

  5. 复制或者下载小红帽的私钥到客户端, 使用它登陆服务器

  6. 最后使用小红帽的私钥登陆服务器, 可以删除小红帽在服务器上的公钥和私钥确保安全

  7. 最后也可以将大灰狼等管理员的公钥, 在服务器上批准使用(重复步骤2), 则可以实现大灰狼不需要小红帽的私钥,用自己的私钥登陆服务器.参考 centos7 搭建 git 服务器

ssh 超时连接中断

可以从客户端和服务器设置定时发送心跳解决, 推荐修改客户端

新版本的 docker 安装可能有部分不同; 遇到问题请查看文章后的参考链接

centos7

  1. 卸载旧版本

  2. 确保安装 docker 库所依赖的包

  3. 设置存储库地址, 注意替换为国内镜像

  4. 安装最新版本 Docker CE

    如果提示接受 GPG 密钥，请验证指纹是否匹配 060A 61C5 1B55 8A7F 742B 77AA C52F EB6B 621E 9F35，如果匹配 ，则接受它。

  5. 安装特定版本的 Docker CE

  6. 启动 Docker

  7. 通过运行 hello-world 映像验证 docker 是否已正确安装

  8. 非root用户身份管理Docker

  9. 配置 Docker 以在启动时启动(可选)

  10. 安装 docker-compose(可选) 如果使用 laradock 则必须安装

ubuntu 16

安装步骤

  1. 卸载旧版本的 docker

  2. 确保安装 docker 库所依赖的包

  3. 设置存储库地址, 注意替换为国内镜像

  4. 安装 Docker CE

  5. 验证是否安装成功(Docker 守护进程将自动启动。)

    如果出现 Hello from Docker! 字样, 则安装成功

可选步骤

  1. 以非 root 用户身份管理 Docker

  2. 将 Docker 配置为在启动时启动

  3. 安装 docker-compose(可选) 如果使用 laradock 则必须安装
